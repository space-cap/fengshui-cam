# Product Requirements Document (PRD)
**프로젝트명:** AI 풍수지리 인테리어 테스트 (가칭: lucky-room)
**문서 버전:** 1.1.0
**작성일:** 2026-04-03
**최종 수정일:** 2026-04-03

---

## 1. 프로젝트 개요 (Overview)
* **목적:** 복잡한 개발 과정 없이 빠르고 가볍게 완성하여 기분 전환 및 성취감을 얻기 위한 퀵(Quick) 프로젝트.
* **핵심 가치:** 최신 멀티모달 AI 기술을 활용하여, 사용자에게 '사진 한 장으로 보는 재미있는 풍수지리 분석'이라는 색다른 경험 제공.
* **타겟 사용자:** 자신의 방이나 자취방 인테리어에 관심이 있고, 가벼운 운세나 심리 테스트를 즐기는 모바일 웹 사용자.

---

## 2. 주요 기능 (Key Features)

### 2.1. 이미지 업로드 (모바일 최적화)
* 사용자가 스마트폰 카메라로 방을 촬영하거나 갤러리에서 사진을 선택하여 업로드.
* **제약 사항:**
  * 업로드 파일 포맷은 이미지(`image/*`)로 제한.
  * 파일 크기 최대 **5MB** 이하로 제한 (클라이언트 사이드에서 사전 검사).
  * 크기 초과 시 업로드 전에 안내 메시지 표시.

### 2.2. AI 풍수지리 분석 및 로딩 처리
* 업로드된 이미지를 Base64 형태로 인코딩하여 OpenAI API로 전송.
* AI 처리 시간(약 3~5초) 동안 사용자 이탈을 막기 위한 직관적인 로딩 UI(스피너 및 텍스트) 제공.
* 로딩 중 버튼 비활성화 처리로 중복 요청 방지.

### 2.3. 결과 출력
* AI가 분석한 다음의 **두 가지** 요소를 화면에 깔끔하게 렌더링:
  1. 전체 풍수 점수 (100점 만점)
  2. 재물운/건강운 상승을 위한 구체적인 인테리어 조언 2가지
* 줄바꿈 등의 텍스트 포맷을 유지하여 가독성 확보.

### 2.4. 에러 처리 (Error Handling)
* **API 호출 실패:** OpenAI API 오류 또는 네트워크 오류 발생 시 사용자에게 친절한 에러 메시지 표시.
* **이미지 미첨부:** 파일 선택 없이 분석 버튼 클릭 시 안내 메시지 표시.
* **파일 형식/크기 오류:** 업로드 전 클라이언트 사이드에서 유효성 검사 후 안내.
* **응답 파싱 실패:** AI 응답이 예상 형식이 아닐 경우 일반 오류 메시지로 대체.

## 3. 사용자 플로우 (User Flow)
1. **[진입]** 메인 웹페이지 접속.
2. **[액션]** '사진 업로드' 영역 터치 -> 카메라 촬영 또는 사진 선택.
3. **[검사]** 파일 크기 및 형식 유효성 자동 검사 (실패 시 안내 메시지 출력 후 중단).
4. **[액션]** '운기 분석 시작하기' 버튼 클릭.
5. **[대기]** 로딩 스피너 및 "AI가 기운을 살피는 중입니다..." 메시지 노출 (이때 버튼은 비활성화).
6. **[결과]** 화면 전환 없이 하단에 AI 분석 결과(점수 및 조언) 출력.
7. **[실패]** API 오류 발생 시 에러 메시지 노출 및 버튼 재활성화.

## 4. 기술 스택 (Tech Stack)
* **Frontend:** HTML5, Vanilla JavaScript, Bootstrap 5 (빠른 UI 구현 및 반응형 레이아웃)
* **Backend:** PHP 8.x (단일 `index.php` 파일로 뷰와 서버 로직 통합 처리)
* **AI API:** OpenAI API (`gpt-4o` 모델) - Vision 기능 활용
* **통신 방식:** PHP `cURL`을 이용한 REST API 호출

---

## 5. AI 프롬프트 가이드라인 (Prompt Design)
* **응답 포맷:** AI에게 항상 **JSON 형식**으로 반환하도록 요청하여 파싱 안정성 확보.
  ```json
  {
    "score": 85,
    "advice": [
      "동쪽 창가에 화분을 놓으면 재물운이 상승합니다.",
      "침대 머리 방향을 북쪽으로 바꾸면 숙면과 건강운에 도움이 됩니다."
    ]
  }
  ```
* **톤 & 매너:** 긍정적이고 재미있는 어투 유지. 과도한 부정적 표현 금지.
* **일관성:** 동일한 이미지에 대해 유사한 결과가 나오도록 시스템 프롬프트 고정.
* **언어:** 결과는 항상 한국어로 반환.

## 6. 보안 및 비용 고려사항 (Security & Cost)
* **API 키 관리:**
  * `.env` 파일 또는 서버 환경 변수로 분리하여 관리.
  * 소스 코드 및 Git 저장소에 API 키 하드코딩 절대 금지.
  * `.gitignore`에 `.env` 파일 추가 필수.
* **요청 제한(Rate Limiting):**
  * 악의적인 대량 요청 방지를 위해 동일 IP의 단시간 내 반복 요청 제한 검토 (Phase 2).
* **API 비용 관리:**
  * `gpt-4o` Vision은 이미지 해상도에 따라 토큰 소모량이 다름.
  * 업로드 이미지를 서버에서 적절한 크기로 리사이즈 후 API 전송 검토.
  * API 사용량 모니터링 및 일일 한도 설정 권장.

---

## 7. 단계별 개발 계획 (Milestones)
* **Phase 1 (MVP - 당일 완성 목표):**
  * 단일 PHP 파일 기반 구현.
  * DB 연동 및 로그인 없이, 접속자 누구나 즉시 테스트 가능하도록 열어둠.
* **Phase 2 (향후 고도화 고려 사항 - Out of Scope for now):**
  * 분석된 사진과 결과를 DB에 저장하여 '나만의 기록' 생성.
  * 카카오톡/URL 공유 기능 추가.
  * AWS S3 등을 활용한 체계적인 이미지 스토리지 관리.
  * IP 기반 Rate Limiting 구현.

## 8. 시스템 아키텍처 (간이)
```text
[Client (Mobile/PC 브라우저)]
   |
   | (0. 파일 크기/형식 유효성 검사 - Client Side)
   |
   | (1. Multipart Form 전송 - 이미지)
   v
[Web Server (PHP - index.php)]
   |
   | (2. 이미지 Base64 인코딩)
   | (3. JSON Payload 생성 - 프롬프트 + 이미지 데이터)
   |
   | (4. cURL 통신 / HTTPS)
   v
[OpenAI API Server (gpt-4o)]
   |
   | (5. 이미지 분석 및 결과 JSON 반환)
   v
[Web Server (PHP - index.php)]
   |
   | (6. JSON 디코딩 및 HTML 렌더링)
   | (7. API 오류 발생 시 에러 처리)
   v
[Client (결과 화면 확인)]
```
